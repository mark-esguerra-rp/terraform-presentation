File System Navigation & Manipulation

Essential commands you'll use daily:

# Navigation and Listing
ls # List files in current directory
ls -la # List all files including hidden, with details
pwd # Print working directory (where am I?)
cd /path # Change directory
 
# File Operations
cp file.txt backup.txt # Copy file
cp -r folder/ backup/ # Copy directory recursively
mv old.txt new.txt # Move/rename file
rm file.txt # Remove file
rm -rf folder/ # Remove directory (careful!)
 
# Viewing Files
cat file.txt # Display entire file
less file.txt # View file with scrolling
head -n 10 file.txt # First 10 lines
tail -f /var/log/app.log # Follow log file in real-time (crucial!)

Real DevOps Scenario

Scenario: You deploy your app and it's not working. You'll need to:
cd /var/log/app
tail -f application.log # Watch logs in real-time
less error.log # Check error logs

Permissions (chmod, chown)

# Understanding permissions
# -rwxr-xr-- means:
# Owner: read, write, execute (rwx)
# Group: read, execute (r-x)
# Others: read only (r--)
 
# Changing permissions
chmod +x script.sh # Make script executable
chmod 755 file.txt # Owner: rwx, Group: rx, Others: rx
chmod 644 config.yml # Owner: rw, Group: r, Others: r
 
# Changing ownership
chown user:group file.txt # Change owner and group
sudo chown root:root app # Change to root (needs sudo)
Real DevOps scenario: Your deployment script fails because it's not executable
chmod +x deploy.sh
./deploy.sh

Process Management

# Viewing processes
ps aux # All running processes
ps aux | grep java # Find Java processes
top # Interactive process viewer (like Task Manager)
htop # Better version of top (if installed)
 
# Controlling processes
kill 1234 # Gracefully stop process with ID 1234
kill -9 1234 # Force kill process
pkill java # Kill all Java processes
Real DevOps scenario: Your app is hanging
ps aux | grep java # Find the process ID
kill 12345 # Try graceful shutdown
kill -9 12345 # Force kill if needed

Package Management (apt)

# Updating package lists
sudo apt update # Update package list
 
# Installing packages
sudo apt install docker.io # Install Docker
sudo apt install nginx # Install Nginx
 
# Removing packages
sudo apt remove nginx # Remove package
sudo apt autoremove # Remove unused dependencies
 
# Searching packages
apt search kubernetes # Find packages
Real DevOps scenario: Setting up a new server
sudo apt update
sudo apt install docker.io git curl

Basic Network Troubleshooting

# Ping: Can I reach this host?
ping 8.8.8.8 # Ping Google DNS
ping google.com # Ping by domain name
 
# Traceroute: What path does traffic take?
traceroute google.com # Shows each hop
 
# Netstat: What ports are open on my machine?
netstat -tuln # List all listening ports
netstat -tuln | grep 8080 # Is port 8080 listening?
 
# Curl: Test HTTP endpoints
curl http://localhost:8080/health
curl -I https://google.com # Just headers
 
# Telnet: Test if port is open
telnet server-ip 22 # Can I reach SSH?

Real Troubleshooting Flow

App won't connect?

ping server-ip # Can I reach the server at all?
telnet server-ip 8080 # Is the port open?
curl http://server-ip:8080 # Can I get HTTP response?

Service won't start?

netstat -tuln | grep 8080 # Is something already on that port?
ps aux | grep java # Is my app actually running?

Slow performance?

traceroute server-ip # Where is the bottleneck?
ping -c 100 server-ip # Check packet loss
