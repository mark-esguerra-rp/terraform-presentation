AWS Infrastructure Management

What is Terraform?

Terraform allows you to build, change, and version infrastructure safely and efficiently.

In simpler terms: codify the process of managing infrastructure, almost like how we write application code.

Key Features

It can execute and automate everything you do in the cloud. It's free and open-source.

Why Infrastructure as Code?

  • ‚Üí Version Control: Track every infrastructure change in Git
  • ‚Üí Repeatability: Create identical environments with confidence
  • ‚Üí Documentation: Your code IS your documentation, always up-to-date
  • ‚Üí Automation: No more manual clicking, reducing human error
  • ‚Üí Collaboration: Team reviews infrastructure changes through pull requests

HashiCorp Configuration Language

Terraform uses HCL to declaratively specify what resources should exist.

resource "aws_s3_bucket" "data_lake" {
bucket = "rocketpartners-data-lake"
region = "us-east-1"
}
# You describe WHAT you want, not HOW to create it

Terraform can manage almost anything across any cloud provider.

Terraform Code Glossary

Terraform is Declarative

You describe WHAT you want to exist, not HOW. Terraform figures out the steps.

Code Structure

  • ‚óè resource: Keyword to create infrastructure
  • ‚óè "aws_s3_bucket": Resource type
  • ‚óè "my_name": Your reference name
  • ‚óè tags: Metadata labels

AWS Terms

  • ‚óè bucket: S3 name (globally unique)
  • ‚óè ami: Amazon Machine Image
  • ‚óè instance_type: Server size
  • ‚óè security_group: Firewall rules (who can access)
  • ‚óè region: AWS data center

Properties

  • ‚óè var.something: Reference variable
  • ‚óè aws_resource.name.id: Reference resources
  • ‚óè ingress/egress: Inbound/outbound rules

Container Terms

  • ‚óè ECR: Elastic Container Registry (Docker images)
  • ‚óè ECS: Elastic Container Service (runs containers)
  • ‚óè Task Definition: Container configuration
