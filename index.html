What is Terraform?

Terraform allows you to build, change, and version infrastructure safely and efficiently.

In simpler terms: codify the process of managing infrastructure, almost like how we write application code.

Key Features It can execute and automate everything you do in the cloud. It's free and open-source.
2/26

Why Infrastructure as Code?

  • Version Control: Track every infrastructure change in Git
  • Repeatability: Create identical environments with confidence
  • Documentation: Your code IS your documentation, always up-to-date
  • Automation: No more manual clicking, reducing human error
  • Collaboration: Team reviews infrastructure changes through pull requests

Terraform Code Glossary

Terraform is Declarative You describe WHAT you want to exist, not HOW. Terraform figures out the steps.

Code Structure

  • resource: Keyword to create infrastructure
  • "aws_s3_bucket": Resource type
  • "my_name": Your reference name
  • tags: Metadata labels

AWS Terms

  • bucket: S3 name (globally unique)
  • ami: Amazon Machine Image
  • instance_type: Server size
  • security_group: Firewall rules (who can access)
  • region: AWS data center

Properties

  • var.something: Reference variable
  • aws_resource.name.id: Reference resources
  • ingress/egress: Inbound/outbound rules

Container Terms

  • ECR: Elastic Container Registry (Docker images)
  • ECS: Elastic Container Service (runs containers)
  • Task Definition: Container configuration

Traditional CMS

Headless CMS

  • Content separate
  • Works everywhere
  • Easy to reuse
